Dear Im\+Gui outputs 16-\/bit vertex indices by default. Allegro doesn\textquotesingle{}t support them natively, so we have two solutions\+: convert the indices manually in \mbox{\hyperlink{imgui__impl__allegro5_8cpp_source}{imgui\+\_\+impl\+\_\+allegro5.\+cpp}}, or compile dear imgui with 32-\/bit indices. You can either modify \mbox{\hyperlink{imconfig_8h_source}{imconfig.\+h}} that comes with Dear Im\+Gui (easier), or set a C++ preprocessor option IMGUI\+\_\+\+USER\+\_\+\+CONFIG to find to a filename. We are providing {\ttfamily \mbox{\hyperlink{imconfig__allegro5_8h_source}{imconfig\+\_\+allegro5.\+h}}} that enables 32-\/bit indices. Note that the backend supports {\itshape BOTH} 16-\/bit and 32-\/bit indices, but 32-\/bit indices will be slightly faster as they won\textquotesingle{}t require a manual conversion.\hypertarget{md___users_cwb_developer_learning_graphic_render_engine_ceres_3rdparty_imgui_examples_example_allegro5__r_e_a_d_m_e_autotoc_md540}{}\doxysection{How to Build}\label{md___users_cwb_developer_learning_graphic_render_engine_ceres_3rdparty_imgui_examples_example_allegro5__r_e_a_d_m_e_autotoc_md540}
