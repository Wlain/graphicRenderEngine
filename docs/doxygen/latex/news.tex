\hypertarget{news_standard_cursors_34}{}\doxysubsubsection{More standard cursors}\label{news_standard_cursors_34}
GLFW now provides the standard cursor shapes \mbox{\hyperlink{group__shapes_gadf2c0a495ec9cef4e1a364cc99aa78da}{GLFW\+\_\+\+RESIZE\+\_\+\+NWSE\+\_\+\+CURSOR}} and \mbox{\hyperlink{group__shapes_gab06bba3b407f92807ba9b48de667a323}{GLFW\+\_\+\+RESIZE\+\_\+\+NESW\+\_\+\+CURSOR}} for diagonal resizing, \mbox{\hyperlink{group__shapes_ga3a5f4811155f95ccafbbb4c9a899fc1d}{GLFW\+\_\+\+RESIZE\+\_\+\+ALL\+\_\+\+CURSOR}} for omni-\/directional resizing and \mbox{\hyperlink{group__shapes_ga297c503095b034bc8891393b637844b1}{GLFW\+\_\+\+NOT\+\_\+\+ALLOWED\+\_\+\+CURSOR}} for showing an action is not allowed.

Unlike the original set, these shapes may not be available everywhere and creation will then fail with the new \mbox{\hyperlink{group__errors_ga09d6943923a70ddef3a085f5baee786c}{GLFW\+\_\+\+CURSOR\+\_\+\+UNAVAILABLE}} error.

The cursors for horizontal and vertical resizing are now referred to as \mbox{\hyperlink{group__shapes_ga2010a43dc1050a7c9154148a63cf01ad}{GLFW\+\_\+\+RESIZE\+\_\+\+EW\+\_\+\+CURSOR}} and \mbox{\hyperlink{group__shapes_gaa59214e8cdc8c8adf08fdf125ed68388}{GLFW\+\_\+\+RESIZE\+\_\+\+NS\+\_\+\+CURSOR}}, and the pointing hand cursor is now referred to as \mbox{\hyperlink{group__shapes_gaad01a50929fb515bf27e4462c51f6ed0}{GLFW\+\_\+\+POINTING\+\_\+\+HAND\+\_\+\+CURSOR}}. The older names are still available.

For more information see \mbox{\hyperlink{input_guide_cursor_standard}{Standard cursor creation}}.\hypertarget{news_mouse_passthrough_34}{}\doxysubsubsection{Mouse event passthrough}\label{news_mouse_passthrough_34}
GLFW now provides the \mbox{\hyperlink{window_guide_GLFW_MOUSE_PASSTHROUGH_hint}{GLFW\+\_\+\+MOUSE\+\_\+\+PASSTHROUGH}} window hint for making a window transparent to mouse input, lettings events pass to whatever window is behind it. This can also be changed after window creation with the matching \mbox{\hyperlink{window_guide_GLFW_MOUSE_PASSTHROUGH_attrib}{window attribute}}.\hypertarget{news_features_34_angle_backend}{}\doxysubsubsection{Support for ANGLE rendering backend selection}\label{news_features_34_angle_backend}
GLFW now provides the \mbox{\hyperlink{intro_guide_GLFW_ANGLE_PLATFORM_TYPE_hint}{GLFW\+\_\+\+ANGLE\+\_\+\+PLATFORM\+\_\+\+TYPE}} init hint for requesting a specific rendering backend when using \href{https://chromium.googlesource.com/angle/angle/}{\texttt{ ANGLE}} to create Open\+GL ES contexts.\hypertarget{news_features_34_win32_keymenu}{}\doxysubsubsection{Support for keyboard access to Windows window menu}\label{news_features_34_win32_keymenu}
GLFW now provides the \mbox{\hyperlink{window_guide_GLFW_WIN32_KEYBOARD_MENU_hint}{GLFW\+\_\+\+WIN32\+\_\+\+KEYBOARD\+\_\+\+MENU}} window hint for enabling keyboard access to the window menu via the Alt+\+Space and Alt-\/and-\/then-\/\+Space shortcuts. This may be useful for more GUI-\/oriented applications.\hypertarget{news_caveats_34}{}\doxysubsection{Caveats for version 3.\+4}\label{news_caveats_34}
\hypertarget{news_joysticks_34}{}\doxysubsubsection{Joystick support is initialized on demand}\label{news_joysticks_34}
The joystick part of GLFW is now initialized when first used, primarily to work around faulty Windows drivers that cause Direct\+Input to take up to several seconds to enumerate devices.

This change will usually not be observable. However, if your application waits for events without having first called any joystick function or created any visible windows, the wait may never unblock as GLFW may not yet have subscribed to joystick related OS events.

To work around this, call any joystick function before waiting for events, for example by setting a \mbox{\hyperlink{input_guide_joystick_event}{joystick callback}}.\hypertarget{news_standalone_34}{}\doxysubsubsection{Tests and examples are disabled when built as a sub-\/project}\label{news_standalone_34}
GLFW now does not build the tests and examples when it is added as a subdirectory of another CMake project. To enable these, set the \mbox{\hyperlink{compile_guide_GLFW_BUILD_TESTS}{GLFW\+\_\+\+BUILD\+\_\+\+TESTS}} and \mbox{\hyperlink{compile_guide_GLFW_BUILD_EXAMPLES}{GLFW\+\_\+\+BUILD\+\_\+\+EXAMPLES}} cache variables before adding the GLFW subdirectory.

\hypertarget{news_initmenu_34}{}\doxysubsubsection{mac\+OS main menu now created at initialization}\label{news_initmenu_34}
GLFW now creates the main menu and completes the initialization of NSApplication during initialization. Programs that do not want a main menu can disable it with the \mbox{\hyperlink{intro_guide_GLFW_COCOA_MENUBAR_hint}{GLFW\+\_\+\+COCOA\+\_\+\+MENUBAR}} init hint.\hypertarget{news_corevideo_34}{}\doxysubsubsection{Core\+Video dependency has been removed}\label{news_corevideo_34}
