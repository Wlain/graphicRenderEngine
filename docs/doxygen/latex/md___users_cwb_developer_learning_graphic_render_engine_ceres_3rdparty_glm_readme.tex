

\href{http://glm.g-truc.net/}{\texttt{ Open\+GL Mathematics}} ({\itshape GLM}) is a header only C++ mathematics library for graphics software based on the \href{https://www.opengl.org/registry/doc/GLSLangSpec.4.50.diff.pdf}{\texttt{ Open\+GL Shading Language (GLSL) specifications}}.

{\itshape GLM} provides classes and functions designed and implemented with the same naming conventions and functionality than {\itshape GLSL} so that anyone who knows {\itshape GLSL}, can use {\itshape GLM} as well in C++.

This project isn\textquotesingle{}t limited to {\itshape GLSL} features. An extension system, based on the {\itshape GLSL} extension conventions, provides extended capabilities\+: matrix transformations, quaternions, data packing, random numbers, noise, etc...

This library works perfectly with {\itshape \href{https://www.opengl.org}{\texttt{ Open\+GL}}} but it also ensures interoperability with other third party libraries and SDK. It is a good candidate for software rendering (raytracing / rasterisation), image processing, physic simulations and any development context that requires a simple and convenient mathematics library.

{\itshape GLM} is written in C++98 but can take advantage of C++11 when supported by the compiler. It is a platform independent library with no dependence and it officially supports the following compilers\+:
\begin{DoxyItemize}
\item \href{https://developer.apple.com/library/mac/documentation/CompilerTools/Conceptual/LLVMCompilerOverview/index.html}{\texttt{ Apple Clang 6.\+0}} and higher
\item \href{http://gcc.gnu.org/}{\texttt{ GCC}} 4.\+7 and higher
\item \href{https://software.intel.com/en-us/intel-compilers}{\texttt{ Intel C++ Composer}} XE 2013 and higher
\item \href{http://llvm.org/}{\texttt{ LLVM}} 3.\+4 and higher
\item \href{http://www.visualstudio.com/}{\texttt{ Visual C++}} 2013 and higher
\item \href{https://developer.nvidia.com/about-cuda}{\texttt{ CUDA}} 7.\+0 and higher (experimental)
\item Any C++11 compiler
\end{DoxyItemize}

For more information about {\itshape GLM}, please have a look at the \mbox{\hyperlink{md___users_cwb_developer_learning_graphic_render_engine_ceres_3rdparty_glm_manual}{manual}} and the \href{http://glm.g-truc.net/0.9.8/api/index.html}{\texttt{ API reference documentation}}.
